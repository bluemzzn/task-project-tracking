
# Evaluation Criteria — NestJS Backend API Project

คะแนนเต็ม: **20 คะแนน**

---

## 1. Code Quality — 3 คะแนน
- โค้ดมีโครงสร้างที่เหมาะสม อ่านง่าย และเป็นระบบ
- ใช้ TypeScript อย่างถูกต้อง (ไม่มี `any`)
- ใช้ Data Type และ Enum ได้เหมาะสม

---

## 2. API Standard — 2 คะแนน
- ใช้ HTTP Method ถูกต้อง (GET, POST, PUT, PATCH, DELETE)
- URL path ตั้งชื่อสื่อความหมาย
- Response format เป็นมาตรฐานเดียวกันทั้งระบบ

---

## 3. Functionality — 4 คะแนน
- CRUD API ทำงานได้ครบตาม requirement
- Endpoint ทุกตัวสามารถใช้งานได้จริง
- ระบบทำงานสอดคล้องกับโจทย์ที่กำหนด

---

## 4. Error Handling & Validation — 4 คะแนน
- มีการ validate ข้อมูลทุก request
- ใช้ HTTP Status Code ได้ถูกต้อง
- จัดการ error ได้เหมาะสม
- ไม่เกิด Error 500 จาก logic ที่สามารถป้องกันได้

> หากพบ Error 500 มากกว่า 5 จุด อาจพิจารณาหักคะแนนในหัวข้อนี้

---

## 5. Documentation — 3 คะแนน
- Swagger API ครบถ้วนทุก Endpoint
- มี UML Diagram ของ Data Model
- เอกสารชัดเจน เป็นระเบียบ

---

## 6. Collaboration & Version Control — 2 คะแนน
- มีการใช้งาน Git อย่างสม่ำเสมอ
- Commit message สื่อความหมาย
- สมาชิกทุกคนมีส่วนร่วม

---

## 7. Presentation & Explanation — 2 คะแนน
- นำเสนอผลงานได้ชัดเจน
- อธิบายโค้ดและแนวคิดการออกแบบได้
- ตอบคำถามเกี่ยวกับระบบของตนเองได้

---

## AI Usage Note
- อนุญาตให้ใช้ AI ช่วยพัฒนาโปรเจค
- นักศึกษาต้องสามารถอธิบายโค้ดของตนเองได้
- หากไม่สามารถอธิบายได้ อาจถูกหักคะแนนในหัวข้อที่เกี่ยวข้อง
